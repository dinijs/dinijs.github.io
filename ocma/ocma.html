Office of Citizenship and Migration Affairs (OCMA) redesign

  • Open website

Office of Citizenship and Migration Affairs (OCMA) is the main authority in Latvia regarding immigration. Their website provides information about passports, visas, residence permits, asylum seeking, immigration statistics, news, etc. This website has vast amounts of information covering wide range of topics. Similarly as other websites faced with complexity, it runs into problem of not providing information in a user friendly manner. The particular section that I have redesigned is resident permit section. I believe that this section has a lot of things that can be improved and I will be outlining the problems and solutions in this case study.

As of November 2023, the website’s user interface has been redesigned to look uniform with other government and municipality websites. They all have same font style, colour scheme and layout structure, which improves the user experience. Moreover, the website has been (finally) adapted for both: mobile and desktop view. The residence permit section has been updated to look and function more in line of the logic outlined in this case study.

Research

Residence Permits page

When a user first lands on the Residence Permits page, he is presented with a lot of information. The page could be divided in 4 parts:

  1. The first part explains what are residence permits;
  2. The second part presents links to actual information about getting a residence permit;
  3. The third part has information about documents that need to be presented upon receiving residence permit;
  4. The fourth part has 13+ links to related information.

The first issue is the enormous amount of information and lack of any direction as to where to start. There are a lot of useful links in the fourth section, but, since they are presented without any order, a user can easily overlook them and later in the process not return back to recheck. Another issue throughout the website is that most of the links take a user outside the website and/or does not present information in a readable manner.


If a user wants to know what documents should be submitted, he or she has to look at the second part and choose between four options:

There are a couple of issues with this part. First of all, the wording of these options is quite long and can be shortened. Second, a user needs to click on a link “stipulated in the list” that will take him outside the website and then review a list of countries to see if his country of citizenship is included. The third issue is that there are links to highly specific information: young Canadian and New Zealand citizen requirements might be useful to some people, but it crowds the page and makes the user experience overwhelming.

Groups

When a user clicks on the “following documents” He or she is presented with a list of different reasons for applying for residence permits. In total there are 31 different groups, some of them very wordy (the longest group description consists of 50 words). The groups can be categorised so that a user does not have to go through all 31 and some of the wording of group names could be simplified as well.

Required documents

When a user clicks on one of 31 groups a list of required documents is provided. However, after viewing the list user still does not know many crucial things:

  1. How long can a user stay in Latvia?
  2. Where can a user submit the documents?
  3. What is the necessary subsistence to stay in Latvia?
  4. What are the next steps after submitting residence permit documents?

Redesign

The problem with the existing resident permit page is lack of a logical path for acquiring the necessary information. This is because the website will display the same information regardless of who is viewing it, as well as display information that is not yet necessary for a user. Thus, very often excess information is visible. The solution to fix this is to introduce a set of variables that would allow for sorting users and redirecting them to different pathways. In order to do this the information has to be gathered, “translated” from the various laws and consolidated from external sources that are already available in OCMA website.

Categories

Before changing the information structure of this section, a crucial task was categorising a list of 31 people groups, so that a user does not have to look through every single option. I was able to create 6 categories: Family relations, Past citizenship, Education, Work or Business, Investing, Other. The method that I used for categorising groups was through interviewing potential users and asking to sort cards with listed groups in stacks depending on their views how the groups would best fit together. On top of that, the group names in those categories were shortened and adjusted based on the information gathered from interviews.

Required documents

The next step was structuring the required documents page in a more user friendly manner. The following information is presented in 4 parts:

  1. The shortened version of group name as well as full group name.
  2. The documents that have to be submitted. The specific embassy where documents have to be submitted changes depending on where a user comes from and the amount of necessary subsistence depends on the selected group.
  3. Information about the maximum period of time a person can receive residence permit.
  4. At the very end user can see what kind of documents he or she has to present to OCMA when receiving residence permit. This information was already present in the Residence permits page, however, since this is the final step of the process, a user will search for this information last.

Information Architecture

By introducing a set of variables the user journey that was a chaotic mess at the beginning (first graph) now is a very straightforward and frustration free experience (second graph).

Variables

The set of variables that were introduced are outlined in the table bellow.


Variable Number of options Description
Embassy 25 Indicates embassy in which user has to submit documents
Passport requirements 5 Indicates whether a user needs to submit statement of punishability
Necessary subsistence 8 Indicates amount of money required to stay in Latvia
Time of stay 15 Indicates the maximum time period of a residence permit

As a bonus, if a user comes from Canada or New Zealand, only he or she will be able to see the special information about travelling from those countries, which reduces information load to users from other countries.
